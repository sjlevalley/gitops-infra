# EC2 Kubernetes Infrastructure

Terraform + Ansible + Helm workflow for provisioning a multi-node Kubernetes cluster on AWS EC2 and bringing up production-grade tooling (GitOps, monitoring) on top of it.

## Architecture

| Node | Role | Default Instance Type |
|------|------|-----------------------|
| `server` | Control plane (API server, etcd, scheduler, controller-manager) | t3.small |
| `node-0` | Worker | t3.small |
| `node-1` | Worker | t3.small |

All nodes run Debian 12 in the AWS default VPC. SSH keys and Ansible inventory are generated by Terraform.

## Repository Structure

```
gitops-infra/
├── cluster-addons/              # Helm-managed cluster-wide services
│   ├── argocd/                  # ArgoCD bootstrap (install instructions)
│   └── monitoring/              # Prometheus + Grafana (values.yaml + ArgoCD Application)
├── cluster-cni-plugins/         # One directory per CNI option
│   ├── calico-kubeadm/          # Terraform + Ansible for a Calico-based cluster
│   ├── cilium-kubeadm/          # Terraform for a Cilium-based cluster
│   └── flannel-kubeadm/         # Terraform + Ansible for a Flannel-based cluster
├── kubernetes-the-hard-way/     # Manual cluster setup (reference / learning)
├── modules/
│   └── kubeadm-ec2-cluster/     # Shared Terraform module (EC2 instances, SG, keys)
└── admin-practice-tasks/        # Kubectl and admin exercises
```

## Prerequisites

| Tool | Purpose | Install |
|------|---------|---------|
| Terraform >= 1.0 | Provision EC2 nodes | [terraform.io/downloads](https://www.terraform.io/downloads) |
| AWS CLI v2 | Authenticate to AWS | [aws.amazon.com/cli](https://aws.amazon.com/cli/) |
| Ansible | Configure nodes and bootstrap kubeadm | `pip install ansible` |
| kubectl | Interact with the cluster | [kubernetes.io/docs/tasks/tools](https://kubernetes.io/docs/tasks/tools/) |
| Helm v3 | Install ArgoCD and cluster addons | [helm.sh/docs/intro/install](https://helm.sh/docs/intro/install/) |

Configure AWS credentials before running Terraform:

```bash
aws configure
```

## Getting Started

### Step 1 — Provision EC2 Nodes with Terraform

Pick a CNI plugin directory and run Terraform inside its `terraform/` subdirectory.

```bash
# Example: Calico
cd cluster-cni-plugins/calico-kubeadm/terraform

terraform init
terraform plan
terraform apply
```

Terraform creates three EC2 instances, generates an SSH key pair (`k8s-key.pem`), and writes an Ansible inventory file. These files are gitignored — they live only on your local machine.

> **Cost note:** `t3.small` instances are not free-tier eligible. Destroy the cluster when not in use (`terraform destroy`) to avoid charges.

### Step 2 — Bootstrap the Kubernetes Cluster

Each CNI plugin directory contains step-by-step setup guides under `setup-steps-post-provisioning/` and, where available, an Ansible playbook that automates the process.

**With Ansible (calico and flannel):**

```bash
# From the terraform/ directory (after terraform apply)
cd ansible
ansible-playbook -i inventory.ini site.yml
```

**Manually (all CNI options):**

Follow the numbered markdown files in `setup-steps-post-provisioning/` in order. These cover:

1. Installing kubeadm, kubelet, and kubectl on all nodes
2. Configuring the container runtime (containerd)
3. Initializing the control plane with `kubeadm init`
4. Installing the CNI plugin
5. Joining worker nodes with `kubeadm join`

Once complete, copy the kubeconfig from the control plane to your local machine:

```bash
scp -i k8s-key.pem ubuntu@<control-plane-ip>:~/.kube/config ~/.kube/config
kubectl get nodes  # should show all three nodes Ready
```

### Step 3 — Install ArgoCD

ArgoCD is the GitOps controller. Install it once via Helm to bootstrap the cluster, then use it to manage all subsequent addons declaratively.

See [`cluster-addons/argocd/README.md`](cluster-addons/argocd/README.md) for full instructions.

```bash
helm repo add argo https://argoproj.github.io/argo-helm
helm repo update

helm install argocd argo/argo-cd \
  --namespace argocd \
  --create-namespace \
  --version ">=7.0.0"
```

### Step 4 — Deploy Cluster Addons

With ArgoCD running, apply the addon Application manifests. ArgoCD pulls each Helm chart and keeps the cluster in sync with the values files committed in this repo.

**Monitoring (Prometheus + Grafana):**

```bash
kubectl apply -f cluster-addons/monitoring/argocd-application.yaml
```

Grafana will be available on NodePort `32000` of any worker node's public IP.
Default login: `admin` / `admin` — change after first login.

See [`cluster-addons/monitoring/README.md`](cluster-addons/monitoring/README.md) for configuration details, direct Helm install instructions, and upgrade/uninstall steps.

## Tearing Down

```bash
cd cluster-cni-plugins/<your-cni>/terraform
terraform destroy
```

This removes all EC2 instances, the security group, and the key pair. The kubeconfig on your local machine will no longer be valid.
